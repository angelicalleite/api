Esta interface permite o acesso aos dados de registros de ocorrências de espécies do explorador do SiBBr. Para realizar consultas à interface, siga as orientações abaixo.

Para maiores informações, por gentileza, acesse a Wiki do projeto no GitHub.

Autenticação e autorização

Para ter acesso aos dados, é necessário ter um usuário cadastrado no sistema com e-mail e senha. Qualquer pessoa pode realizar o cadastro aqui.

Após o cadastro, o usuário é criado no sistema e permanece como um usuário não autorizado até que um administrador autorize o seu acesso através das operações administrativas.

Os usuários podem se identificar no sistema através da página de login.

Ao se identificarem corretamente, recebem um token válido, que deve ser utilizado em todas as chamadas a consultas de dados da API.

Este token não deve ser compartilhado com outros usuários, uma vez que representa a identificação do acesso autorizado do usuário ao consumo dos dados.

A restrição de acesso às operações se aplica a ambos o usuário comum e o usuário administrador.

O token possui uma validade de 7 dias a partir de sua criação. Após este período, ele perde a validade e o usuário recebe a notificação de que precisa realizar um novo login, para receber um novo token e poder seguir realizando consultas.

Operações

  • 1. /admin - Operações administrativas: Gestão de usuários
  • 2. /login - Login: Identificação no sistema para visualização do token necessário para acesso aos dados
  • 3. /registrar - Registrar no sistema: Usuários podem criar contas para utilizar o sistema
  • 4. /alterarSenha - Alteração de senhas: Usuários podem alterar as senhas das suas contas
  • 5. /recursos - Recursos: Listagem de recursos, busca de ocorrências por recurso
  • 6. /ocorrencia - Registros de ocorrência: Busca por registros de ocorrência com aplicação de filtros
  • 7. /estatisticas - Estatísticas: Visualização de estatísticas dos dados

This interface allows access to the occurrence records data from SiBBr's explorer. To make queries to the interface, follow the instructions bellow.

For further information, please, accesse the project Wiki on GitHub.

Authentication and authorization

To have access to data, it is required to have a registered user in the system with e-mail and password. Any person can register here.

After registration, the user is created in the system and remains as an unauthorized user until the administer grants access to the account through the administrative operations.

Users can log onto the system through the login page.

After proper login, the accounts receive a valid token, that must be used in all calls to API queries.

This token must not be shared with other users, since it represents the identification of the users's authorized access to data consumption.

The access restriction to the operations applies to both the regular and administer users.

The token is valid for 7 days from its creation. After that, expires and the user receives a notification about having to login again in order to receive a new token and be able to resume queries.

Operations

  • 1. /admin - Aadministrative operations: User management
  • 2. /login - Login: System logon to visualize the data access token
  • 3. /registrar - Register on the system: Users may create accounts to use the system
  • 4. /alterarSenha - Password change: Users may change their account password
  • 5. /recursos - Resources: Resource list, occurrence search by resource
  • 6. /ocorrencia - Ocurrence records: Search for occurrence records with filters
  • 7. /estatisticas - Statistics: Data visualization statistics
